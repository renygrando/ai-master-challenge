You are a senior full-stack engineer and AI systems architect.

Your task is to build a production-ready SaaS ticketing platform called:

HERMES

Hermes is an AI-native IT support ticket system designed to detect operational bottlenecks, classify tickets automatically, and route them intelligently.

The system focuses on solving the biggest inefficiencies in support operations:

• diagnosis delays
• escalations
• approval workflows
• ticket backlog
• repetitive requests

Hermes must be built with modern architecture and optimized for AI-driven workflows.

AI SYSTEM

Hermes AI consists of three models.

--------------------------------------------------

1 TICKET CLASSIFIER

Input:
ticket text

Output:

category
confidence_score

Categories:

Hardware
Access
HR Support
Storage
Purchase
Administrative rights
Internal Project
Misc

Implementation:

Use transformer model:

sentence-transformers/all-MiniLM-L6-v2

Classification pipeline:

text embedding
similarity search
classification layer

Example response:

{
 category: "Access",
 confidence: 0.92
}

--------------------------------------------------

2 SIMILAR TICKET ENGINE

Use vector search.

Process:

1 embed ticket text
2 search FAISS index
3 retrieve top 5 similar tickets
4 extract solutions
5 generate suggested response

--------------------------------------------------

3 RESPONSE SUGGESTION AI

Prompt template:

"You are a senior IT support engineer.
Generate a helpful response for this ticket using past resolutions."

Input:

ticket
similar tickets
knowledge base

Output:

suggested response
confidence score

--------------------------------------------------

ROUTING ENGINE

Every ticket must be routed automatically.

Routing rules:

if category = Access
route to Access Automation Engine

if category = Hardware
route to Hardware Team

if category = HR Support
route to HR Support Team

Priority rules:

if customer impact high
priority = High

if SLA risk detected
priority escalates

--------------------------------------------------

AGING SYSTEM

Tickets automatically increase priority over time.

Rules:

Low → Medium after 6 hours
Medium → High after 12 hours

--------------------------------------------------

AUTO RESOLUTION SYSTEM

Hermes automatically resolves some tickets.

Example:

Access requests

Workflow:

AI verification
policy check
API call to authentication system

Ticket resolved automatically.
